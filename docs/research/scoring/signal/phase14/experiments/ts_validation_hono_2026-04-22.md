## §1 Calibration

**Target N_CAL:** 500 (per template)  
**Actual N_CAL used:** 485

**Why the adjustment:** The Hono source tree has 196 non-test `.ts`/`.tsx` source files. An initial sampler that only looked for `function_declaration` / `class_declaration` / `interface_declaration` / `type_alias_declaration` nodes yielded only 206 qualifying hunks, because Hono exports the vast majority of its logic as `export const foo = () => {...}` (arrow function consts). 116/196 files had zero qualifying hunks under the initial sampler.

The sampler was extended to also extract `variable_declarator` nodes where the RHS is an `arrow_function` or `function_expression`. With this fix the pool grew to **488–494 hunks** (varies across PR pre-merge snapshots). N_CAL was reduced from 500 to **485** to stay safely below the minimum pool size of 488.

The difference between 485 and 500 calibration hunks is negligible — both are close to the pool size and the max-threshold (`threshold_percentile=None`) is dominated by the handful of highest-BPE hunks regardless.

**Threshold stability:** BPE thresholds are remarkably stable across all 5 PR snapshots:

| PR | pre_sha | bpe_threshold |
|----|---------|---------------|
| #4883 | 3779927c | 5.8628 |
| #4848 | 75b43084 | 5.8514 |
| #4750 | 488ea6ab | 5.8491 |
| #4780 | 0e80acb9 | 5.8487 |
| #4834 | 9f374a55 | 5.8538 |

Range: 5.8487–5.8628 (Δ = 0.014). This is much tighter than the Python corpora, which showed more variance. Likely because the Hono codebase evolves steadily without dramatic feature-density shifts.

Neither exclusion is a scorer regression — the `is_data_dominant` and `is_auto_generated` methods work correctly on their intended cases (locale data files, scaffolded generated code). These are heuristic false positives at the boundary of their intended domain.

**Hono has no locale files** — the TypeScript data-dominance concern from the template (aimed at faker-js locale tables) does not apply here. No `src/locales/**` or equivalent exists.

**Scorer behaves sensibly on the Hono corpus.**

Key observations:

1. **0% source flag rate.** 22 source hunks across 5 PRs, zero flags. The scorer correctly passes all standard fix/feat hunks through without false positives.

2. **All test flags are INTENTIONAL_STYLE_INTRO.** The 3 test flags each correspond to new test suites for new features in the same PR — exactly the kind of signal we'd expect the scorer to potentially flag (new patterns not yet in the corpus) but that aren't style violations.
