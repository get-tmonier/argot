# Language Adapter Seam + TypeScript Adapter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split all language-specific logic out of the Phase 14 scorer into a `LanguageAdapter` protocol, move Python logic behind a `PythonAdapter`, implement a functional `TypeScriptAdapter`, and wire both into the scorer via a registry — behavior-preserving for Python, ready for TS integration validation.

**Architecture:** A `LanguageAdapter` protocol lives in a new `adapters/` package alongside the scorer. The `SequentialImportBpeScorer` resolves the appropriate adapter from a registry based on the extensions found in `model_a_files`. All Python-specific calls (AST parsing, `is_data_dominant`, `is_auto_generated`, prose masking) are delegated to the adapter. The existing `filters/` shims are deprecated but not removed.

**Tech Stack:** Python 3.13, tree-sitter 0.23.2, tree-sitter-python 0.23.6 (already used), tree-sitter-typescript 0.23.2 (already in pyproject.toml), pytest, mypy strict.

## Key Design Decisions (locked in)

1. **`prose_line_ranges` returns `frozenset[int]`** — the spec signature says `list[tuple[int, int]]` but the scorer iterates line numbers directly. Using `frozenset[int]` keeps all 29+ scorer tests passing without any scorer logic change. Noted in §3 of the report.

2. **`extract_imports` filters relative paths** — both adapters return only non-relative module specifiers. Relative imports (`./foo`, `../bar`) are always repo-internal and are excluded during extraction. The `ImportGraphScorer` comparison (`hunk_imports - _repo_modules`) works correctly without extra logic.

3. **`resolve_repo_modules` is implemented but not wired into `ImportGraphScorer.fit()`** — the spec defines it for the TS adapter (package.json, tsconfig paths), but wiring it requires adding `repo_root: Path` to the scorer interface, which would change the existing API. This is a Task 11 TODO in the report.

4. **Adapter is resolved once at `SequentialImportBpeScorer.__init__`** — derived from the majority file extension in `model_a_files`. All Python tests pass `.py` files → PythonAdapter → identical behavior.

5. **`TypeScriptAdapter.is_data_dominant`** — content-based only (no path filters). Checks if top-level `const`/`let`/`export const` where the initializer is an `array` or `object` literal covers ≥65% of non-blank lines. Must return `True` for faker-js locale-style files.

from pathlib import Path
from typing import Protocol, runtime_checkable


@runtime_checkable
class LanguageAdapter(Protocol):
    """Pluggable language-specific logic for import extraction, filtering, and prose masking.

    All methods are pure (no I/O) except ``resolve_repo_modules`` which reads the repo
    on disk.  Implementations must be safe to call on syntactically invalid fragments
    (e.g. mid-block hunk slices) — return empty sets / False gracefully.
    """

    file_extensions: frozenset[str]

    def extract_imports(self, source: str) -> set[str]:
        """Return top-level (non-relative) module specifiers imported in *source*.

        Relative paths (``./foo``, ``../bar``) are always repo-internal and must NOT
        be returned.  On parse error return an empty set — never raise.
        """
        ...

    def resolve_repo_modules(self, repo_root: Path) -> set[str]:
        """Return the set of known internal module names for *repo_root*.

        For Python: returns an empty set (internal modules are discovered via
        ``extract_imports`` over model_A files during ``ImportGraphScorer.fit``).
        For TypeScript: reads ``package.json`` (package name + workspace names) and
        ``tsconfig.json`` ``compilerOptions.paths`` (alias prefixes).
        """
        ...

    def is_data_dominant(self, source: str, threshold: float = 0.65) -> bool:
        """Return True if *source* is overwhelmingly composed of static data literals.

        Must trigger on locale-style files (arrays of strings / objects in export const
        form) regardless of file path — content-based only, no path heuristics.
        """
        ...

    def is_auto_generated(self, source: str) -> bool:
        """Return True if *source* contains an auto-generation marker in its header.

        Only inspects comment nodes (not string literals) to avoid false positives.
        """
        ...

    def prose_line_ranges(self, source: str) -> frozenset[int]:
        """Return 1-indexed line numbers that are pure prose (comments / docstrings).

        Used to blank prose before BPE scoring so natural-language tokens do not
        inflate the score.  On parse error return an empty frozenset.

# engine/argot/research/signal/phase14/adapters/registry.py
"""Registry mapping file extensions to LanguageAdapter singleton instances."""

from __future__ import annotations

from argot.research.signal.phase14.adapters.language_adapter import LanguageAdapter
from argot.research.signal.phase14.adapters.python_adapter import PythonAdapter

# Lazy-initialised singleton adapters (avoids loading tree-sitter grammars at import time)
_adapters: dict[str, LanguageAdapter] = {}


def _python() -> LanguageAdapter:
    if ".py" not in _adapters:
        _adapters[".py"] = PythonAdapter()
    return _adapters[".py"]


def _typescript() -> LanguageAdapter:
    if ".ts" not in _adapters:
        from argot.research.signal.phase14.adapters.typescript_adapter import TypeScriptAdapter
        _adapters[".ts"] = TypeScriptAdapter()
        _adapters[".tsx"] = _adapters[".ts"]
    return _adapters[".ts"]


_EXTENSION_MAP: dict[str, str] = {
    ".py": "python",
    ".ts": "typescript",
    ".tsx": "typescript",
}

_FACTORIES = {
    "python": _python,
    "typescript": _typescript,
}


def get_adapter(ext: str) -> LanguageAdapter:
    """Return the adapter for file extension *ext* (e.g. ``".py"``).

    Falls back to PythonAdapter for unknown extensions.
    """
    lang = _EXTENSION_MAP.get(ext, "python")
    return _FACTORIES[lang]()


def adapter_for_files(paths: list[str]) -> LanguageAdapter:
    """Resolve the dominant adapter from a list of file path strings.

    Counts extensions, picks the most common one that has a registered adapter,
    falls back to PythonAdapter.
    """
    from collections import Counter

    counts: Counter[str] = Counter()
    for p in paths:
        ext = p[p.rfind("."):] if "." in p else ""
        if ext in _EXTENSION_MAP:
            counts[ext] += 1

    if not counts:
        return get_adapter(".py")
    dominant_ext = counts.most_common(1)[0][0]
    return get_adapter(dominant_ext)
```

def _strip_quotes(text: str) -> str:
    """Strip surrounding quotes from a tree-sitter string node text."""
    if len(text) >= 2 and text[0] in ('"', "'", "`") and text[-1] == text[0]:
        return text[1:-1]
    return text


def _is_relative(specifier: str) -> bool:
    return specifier.startswith("./") or specifier.startswith("../")


def _extract_require_imports(root: Node) -> set[str]:
    """Walk AST for require('specifier') call expressions."""
    mods: set[str] = set()
    for node in _walk(root):
        if node.type != "call_expression":
            continue
        fn = node.child_by_field_name("function")
        args = node.child_by_field_name("arguments")
        if fn is None or args is None:
            continue
        if fn.type != "identifier" or (fn.text or b"").decode("utf-8") != "require":
            continue
        # First argument should be a string
        for arg in args.children:
            if arg.type == "string" and arg.text:
                spec = _strip_quotes(arg.text.decode("utf-8"))
                if spec and not _is_relative(spec):
                    mods.add(spec)
    return mods


def _walk(node: Node):  # type: ignore[return]
    """Depth-first traversal yielding every node."""
    yield node
    for child in node.children:
        yield from _walk(child)
```

class TypeScriptAdapter:
    """LanguageAdapter implementation for TypeScript (.ts) and TSX (.tsx) source files."""

    file_extensions: frozenset[str] = frozenset({".ts", ".tsx"})

    def _get_parser_and_queries(
        self, extension: str
    ) -> tuple[TsParser, object, object]:
        if extension == ".tsx":
            parser = TsParser(_TSX_LANGUAGE)
            import_q = _IMPORT_QUERY_TSX
            comment_q = _COMMENT_QUERY_TSX
        else:
            parser = TsParser(_TS_LANGUAGE)
            import_q = _IMPORT_QUERY_TS
            comment_q = _COMMENT_QUERY_TS
        return parser, import_q, comment_q

    def extract_imports(self, source: str, extension: str = ".ts") -> set[str]:
        """Return non-relative module specifiers imported in *source*.

        Handles ``import ... from 'specifier'``, ``import 'specifier'`` (side-effect),
        and ``require('specifier')`` calls.  Relative imports are excluded.
        Returns empty set on parse error.
        """
        try:
            parser, import_q, _ = self._get_parser_and_queries(extension)
            tree = parser.parse(source.encode("utf-8"))
            captures: dict[str, list[Node]] = import_q.captures(tree.root_node)  # type: ignore[attr-defined]
            mods: set[str] = set()
            for node in captures.get("src", []):
                if node.text:
                    spec = _strip_quotes(node.text.decode("utf-8"))
                    if spec and not _is_relative(spec):
                        mods.add(spec)
            mods |= _extract_require_imports(tree.root_node)
            return mods
        except Exception:
            return set()

    def resolve_repo_modules(self, repo_root: Path) -> set[str]:
        """Return package names and tsconfig path alias prefixes for *repo_root*.

        Reads ``package.json`` (own name + workspace names) and
        ``tsconfig.json`` ``compilerOptions.paths``.  Falls back gracefully if
        either file is absent or malformed.
        """
        mods: set[str] = set()

        pkg_json = repo_root / "package.json"
        if pkg_json.exists():
            try:
                data: dict[str, object] = json.loads(pkg_json.read_text(encoding="utf-8"))
                if name := data.get("name"):
                    mods.add(str(name))
                workspaces = data.get("workspaces", [])
                if isinstance(workspaces, dict):
                    workspaces = workspaces.get("packages", [])
                if isinstance(workspaces, list):
                    for pattern in workspaces:
                        if isinstance(pattern, str):
                            for ws_pkg in repo_root.glob(str(pattern) + "/package.json"):
                                try:
                                    ws_data: dict[str, object] = json.loads(
                                        ws_pkg.read_text(encoding="utf-8")
                                    )
                                    if ws_name := ws_data.get("name"):
                                        mods.add(str(ws_name))
                                except Exception:
                                    pass
            except Exception:
                pass

        tsconfig = repo_root / "tsconfig.json"
        if tsconfig.exists():
            try:
                tsdata: dict[str, object] = json.loads(tsconfig.read_text(encoding="utf-8"))
                compiler_opts = tsdata.get("compilerOptions", {})
                if isinstance(compiler_opts, dict):
                    paths = compiler_opts.get("paths", {})
                    if isinstance(paths, dict):
                        for alias in paths:
                            # "@scope/pkg/*" → "@scope/pkg"
                            prefix = str(alias).rstrip("/*").rstrip("/")
                            if prefix:
                                mods.add(prefix)
            except Exception:
                pass

        return mods

    def is_data_dominant(self, source: str, threshold: float = 0.65) -> bool:
        """Return True if the file is overwhelmingly composed of top-level data literals.

        Walks the tree-sitter AST (TS grammar) looking for top-level
        ``lexical_declaration`` / ``export_statement`` nodes where the
        ``variable_declarator`` value is an ``array`` or ``object`` literal.
        Computes ``data_literal_lines / total_nonblank_lines`` and compares
        against *threshold*.
        """
        if not source or not source.strip():
            return False
        try:
            parser = TsParser(_TS_LANGUAGE)
            tree = parser.parse(source.encode("utf-8"))
            root = tree.root_node
        except Exception:
            return False

        total_nonblank = sum(1 for line in source.splitlines() if line.strip())
        if total_nonblank == 0:
            return False

        data_rows: set[int] = set()
        _collect_ts_data_rows(root, data_rows)
        ratio = len(data_rows) / total_nonblank
        return ratio > threshold

    def is_auto_generated(self, source: str, head_lines: int = 40) -> bool:
        """Return True if the file header contains an auto-generation marker in a comment.

        Only inspects tree-sitter ``comment`` nodes (both ``//`` and ``/* */``).
        Falls back to a line-prefix heuristic if parsing fails.
        """
        if not source:
            return False
        try:
            parser = TsParser(_TS_LANGUAGE)
            tree = parser.parse(source.encode("utf-8"))
            captures: dict[str, list[Node]] = _COMMENT_QUERY_TS.captures(tree.root_node)
            comment_nodes: list[Node] = captures.get("c", [])

            for node in comment_nodes:
                line_index = node.start_point[0]
                if line_index >= head_lines:
                    continue
                text = node.text.decode("utf-8") if node.text else ""
                ll = text.lower()
                if line_index < 10 and any(m in ll for m in _STRONG_FIRST_10_MARKERS):
                    return True
                if any(m in ll for m in _ALL_MARKERS):
                    return True
            return False
        except Exception:
            lines = source.splitlines()
            head = lines[:head_lines]
            for line in head[:10]:
                stripped = line.lstrip()
                if not (stripped.startswith("//") or stripped.startswith("/*")):
                    continue
                ll = line.lower()
                if any(m in ll for m in _STRONG_FIRST_10_MARKERS):
                    return True
            for line in head:
                stripped = line.lstrip()
                if not (stripped.startswith("//") or stripped.startswith("/*")):
                    continue
                ll = line.lower()
                if any(m in ll for m in _ALL_MARKERS):
                    return True
            return False

    def prose_line_ranges(self, source: str, extension: str = ".ts") -> frozenset[int]:
        """Return 1-indexed line numbers covered by comments (line and block).

        TypeScript has no docstring concept; all prose is in comments.
        """
        try:
            parser, _, comment_q = self._get_parser_and_queries(extension)
            tree = parser.parse(source.encode("utf-8"))
            captures: dict[str, list[Node]] = comment_q.captures(tree.root_node)  # type: ignore[attr-defined]
            rows: set[int] = set()
            for node in captures.get("c", []):
                # Comment node spans start_point to end_point (0-indexed rows)
                start = node.start_point[0] + 1  # 1-indexed
                end = node.end_point[0] + 1
                rows.update(range(start, end + 1))
            return frozenset(rows)
        except Exception:
            return frozenset()
```

"""Import-graph foreign-module scorer.

Counts the number of top-level modules imported inside a hunk that were
never seen in the repo's own source (model_A).  Score 0 = no foreign
imports; score ≥ 1 = paradigm-break signal.

Design decisions:
- Top-level module only: ``from sqlalchemy.orm import Session`` → "sqlalchemy".
- Repo modules include all third-party deps seen in model_A.  Scorer only flags
  modules *never* encountered in model_A.
- Stdlib has no special treatment: if the repo never imports ``threading``,
  a hunk importing ``threading`` is flagged.
- Relative imports (``from . import X``, ``from ..foo import Y``) are ignored.
- Hunk parsing: if parsing raises an error (mid-block slice),
  ``adapter.extract_imports`` returns an empty set.  Callers that need to detect
  imports from the file header must pass them separately — see
  ``SequentialImportBpeScorer.score_hunk``.
"""

from __future__ import annotations

import ast
import re
from collections.abc import Iterable
from pathlib import Path

from argot.research.signal.phase14.adapters.language_adapter import LanguageAdapter
from argot.research.signal.phase14.adapters.registry import get_adapter

# Matches ``import foo`` or ``import foo.bar`` (captures the leading name)
_RE_IMPORT = re.compile(r"^\s*import\s+([A-Za-z_]\w*)", re.MULTILINE)
# Matches ``from foo`` or ``from foo.bar`` but NOT ``from .foo`` (relative)
_RE_FROM_IMPORT = re.compile(r"^\s*from\s+([A-Za-z_]\w*)", re.MULTILINE)


def _top_level(module: str) -> str:
    return module.split(".")[0]


def _imports_from_ast(source: str) -> set[str]:
    try:
        tree = ast.parse(source)
    except SyntaxError:
        # Mid-block hunk slices are not valid Python — return empty set.
        # Do NOT fall back to regex: regex matches module names from prose/docstrings
        # and produces false positives.
        return set()

    modules: set[str] = set()
    for node in ast.walk(tree):
        if isinstance(node, ast.Import):
            for alias in node.names:
                modules.add(_top_level(alias.name))
        elif isinstance(node, ast.ImportFrom):
            if node.level and node.level > 0:
                continue  # relative import — skip
            if node.module:
                modules.add(_top_level(node.module))
    return modules


def _imports_from_regex(source: str) -> set[str]:
    modules: set[str] = set()
    for m in _RE_IMPORT.finditer(source):
        modules.add(m.group(1))
    for m in _RE_FROM_IMPORT.finditer(source):
        modules.add(m.group(1))
    return modules


class ImportGraphScorer:
    """Score a hunk by how many modules it imports that are foreign to the repo."""

    def __init__(self, adapter: LanguageAdapter | None = None) -> None:
        self._adapter: LanguageAdapter = adapter if adapter is not None else get_adapter(".py")
        self._repo_modules: frozenset[str] = frozenset()

    def fit(self, model_a_files: Iterable[Path]) -> None:
        """Parse every file in model_A and collect the set of top-level modules."""
        seen: set[str] = set()
        for path in model_a_files:
            try:
                source = path.read_text(encoding="utf-8", errors="replace")
            except OSError:
                continue
            seen.update(self._adapter.extract_imports(source))
        self._repo_modules = frozenset(seen)

    def score_hunk(self, hunk_source: str) -> float:
        """Return the count of top-level modules in hunk_source not seen in model_A.

        Returns 0.0 if no foreign imports are found.
        """
        hunk_modules = self._adapter.extract_imports(hunk_source)
        foreign = hunk_modules - self._repo_modules
        return float(len(foreign))
```

## §1 — Design Decisions

### Module-specifier resolution strategy for TypeScript

`TypeScriptAdapter.extract_imports` returns the raw module specifier string as written
in the source (e.g. `"react"`, `"@scope/pkg"`, `"lodash"`).  Relative imports
(`"./utils"`, `"../types"`) are filtered at extraction time — they are always
repo-internal by definition, so the `ImportGraphScorer` comparison
(`hunk_imports - _repo_modules`) works correctly without any additional logic.

`resolve_repo_modules` reads `package.json` (own name + workspace package names) and
`tsconfig.json` `compilerOptions.paths` (alias prefixes).  This supplements
`_repo_modules` built from `fit()` (which unions all import specifiers seen across
model_A files).

### Data-dominance heuristic port

The Python heuristic (ratio of lines covered by list/tuple/dict/set RHS assignments)
maps directly to TypeScript via the tree-sitter TS grammar:

- Python: `assignment` with `list | tuple | dictionary | set` RHS
- TypeScript: `variable_declarator` inside `lexical_declaration | export_statement`
  with `array | object` RHS

The threshold (0.65) and ratio computation are identical.  This correctly triggers on
faker-js locale files (e.g. `export const firstName = ['Alice', 'Bob', ...300 more...]`).

### `prose_line_ranges` return type deviation

The spec protocol signature is `list[tuple[int, int]]` (line ranges).  The
implementation uses `frozenset[int]` (individual 1-indexed line numbers) to match
the existing `Parser` protocol and keep `SequentialImportBpeScorer`'s prose-masking
logic unchanged (it iterates line numbers directly).  This is a minor signature
deviation noted for future cleanup.

## §3 — Known Limitations / Follow-up

1. **`resolve_repo_modules` not wired into `ImportGraphScorer.fit()`** — The method is
   implemented on both adapters but `ImportGraphScorer.fit(model_a_files)` still builds
   `_repo_modules` from per-file `extract_imports` calls, not from `resolve_repo_modules`.
   Wiring this requires adding `repo_root: Path` to the scorer's constructor, which
   changes the external API.  Planned for the TS integration validation step.

2. **Hunk-scope Stage 1 still uses `_imports_from_ast` for Python** — When
   `file_source is not None`, `score_hunk` calls `_imports_from_ast(hunk_content)`
   (Python AST) directly rather than `self._adapter.extract_imports(hunk_content)`.
   This is correct for Python (identical behavior) but means the TS hunk-scope path
   is not yet wired.  Fix in TS integration step.

3. **Dynamic `import()` calls not handled** — `TypeScriptAdapter.extract_imports`
   handles `import ... from` and `require()` but not `await import('specifier')`.
   Dynamic imports are uncommon in the training corpora but should be added before
   the TS scorer is considered complete.

4. **`.d.ts` declaration files** — These are TypeScript but contain no runnable logic.
   The `TypeScriptAdapter` does not special-case them.  `is_auto_generated` will catch
   most generated `.d.ts` files via header comments, but hand-authored declaration
   files could still affect the BPE corpus.  Flag for the eligibility check command.

5. **JSDoc `@param` / `@returns` are comment lines** — `prose_line_ranges` counts all
   comment content as prose.  For JSDoc blocks this is correct but means a
   documentation-heavy file might have many blanked lines in BPE scoring.  Acceptable
   for V0; revisit if BPE scores on well-documented TS code are unexpectedly low.
